# Prioridade (impacto x esforco)

## Alto impacto / Baixo esforco (quick wins)
- Filtro por nome (Vagas / Candidaturas).
- Filtro por contrato + nome (Documentos).
- Bug: status "Concluido" redirecionando (Documentos).
- Bug: retorno para pasta "Pessoal" apos solicitacao (Documentos).
- Buscar por nome em Gerenciar Usuarios.

## Alto impacto / Medio ou alto esforco (criticos)
- Modelo SaaS (multiempresa, planos, modulos, bloqueios).
- Folha de ponto (calculos, relatorios, banco de horas, escala).
- Contracheques (pagina exclusiva + envio em massa + CSV).
- Registro de sobreaviso (colaborador + RH + relatorios).
- Medidas de seguranca de dados/documentos.

## Medio impacto / Medio esforco
- Pagina inicial RH (avisos, pendencias, popup).
- Pagina inicial Colaborador (avisos, popups).
- Filtros e exibicoes no perfil do candidato.
- Marcar colaborador como desligado + aba especifica.
- Excluir colaborador sem vinculos (com motivo).
- Restricao de upload para PDF (RH/Colaborador).
- Pagina de vagas flexivel (integracao/standalone).

## Medio/baixo impacto / Alto esforco (longo prazo)
- Reescrita de conteudo para reduzir dependencia de imagens (SEO/acessibilidade).
- Migracao completa de build ou refatoracoes estruturais amplas.

# Dependencias entre itens

## Macro
- SaaS/multiempresa/planos/modulos depende de:
  - Definicao de tenant/empresa no banco e no login.
  - Isolamento de dados por empresa.
  - Camada de permissao por modulo.

## Folha de ponto
- Calculos de horas extras/noturnas dependem de:
  - Escala de trabalho cadastrada.
  - Regras de feriados/recessos.
  - Registro consistente de ponto (colaborador e ajustes).
- Relatorios dependem de:
  - Dados de ponto consolidados.
  - Regras de calculo e aprovacao.

## Sobreaviso
- Registro de sobreaviso (colaborador) depende de:
  - Estrutura de ponto para tipos de registro.
  - Permissoes e fluxo de aprovacao no RH.
- Relatorio de sobreaviso depende de:
  - Registros aprovados e regras de calculo.

## Contracheques
- Envio em massa e CSV dependem de:
  - Modelo de contratos por colaborador.
  - Padrao de importacao e validacao de dados.

## Documentos
- Restricao de upload para PDF depende de:
  - Validacao MIME/extension no backend.
  - Feedback de erro no frontend.

# Estimativa de horas/etapas (macro)

## SaaS (multiempresa/planos/modulos)
- Etapas: modelagem tenant -> permissao por modulo -> billing (se aplicavel) -> ajustes no admin.
- Estimativa: 120h a 200h.

## Folha de ponto (RH + Colaborador + calculos + relatorios)
- Etapas: UX fluxo ponto -> regras de escala/feriados -> calculos -> aprovacao -> relatorios.
- Estimativa: 160h a 240h.

## Sobreaviso (completo)
- Etapas: modelo de dados -> registro colaborador -> aprovacao RH -> relatorios.
- Estimativa: 60h a 100h.

## Contracheques (pagina + massa + CSV)
- Etapas: pagina dedicada -> upload em massa -> importacao CSV -> validacao/feedback.
- Estimativa: 60h a 120h.

## Documentos (filtros + bugs + PDF only)
- Etapas: filtros combinados -> correcoes de fluxo -> validacao upload.
- Estimativa: 24h a 40h.

## Vagas/Candidatos/Usuarios (filtros e exibicoes)
- Etapas: filtros e exibicoes no perfil -> organizacao por abas.
- Estimativa: 16h a 32h.

## Colaboradores (desligar/excluir)
- Etapas: regras de exclusao -> auditoria/motivo -> bloqueio de acesso.
- Estimativa: 20h a 40h.

## Paginas iniciais (RH + Colaborador)
- Etapas: mural de avisos -> popup de documentos/contracheques -> pendencias.
- Estimativa: 24h a 48h.

## Seguranca (dados/documentos)
- Etapas: politica de acesso -> mascaramento/permicoes -> auditoria basica.
- Estimativa: 40h a 80h.
