FLUXO DE TRABALHO OBRIGATÓRIO
==================================================

ETAPA 1 — Briefing e Arquitetura
- Eu fornecerei um briefing inicial do projeto.
- A partir dele, você deve:
  a) Entender objetivos, usuários e contexto de negócio
  b) Identificar contextos técnicos:
     - Web, Mobile, API, Supabase, RLS
     - three.js (renderização 3D, cena, performance)
  c) Definir entidades, modelo de dados e multitenancy (se houver)
  d) Definir fluxo de dados (client → server → DB)
  e) Definir como o three.js será usado:
     - visualização
     - interação
     - performance e fallback
- Em seguida, você deve criar e preencher o arquivo arquitetura.md.

ETAPA 2 — Validação (OBRIGATÓRIA)
- Antes de preencher agent.md ou design-system.md:
  a) Faça TODAS as perguntas necessárias
  b) Não presuma nada
  c) Não avance enquanto houver dúvidas

As perguntas devem cobrir, no mínimo:
- Escopo do 3D (decorativo, funcional, crítico)
- WebGL target (desktop, mobile, low-end)
- SSR vs CSR
- Estratégia de loading/fallback para three.js
- Uso de Supabase (RLS, multitenancy)
- API pública ou privada
- Dados sensíveis / LGPD
- Versionamento inicial
- Requisitos visuais e identidade

ETAPA 3 — Preenchimento dos Documentos
- Somente após TODAS as dúvidas sanadas:
  a) Preencher agent.md
  b) Definir versões iniciais
  c) Marcar contextos ativos
  d) Preencher design-system.md (se houver UI)
  e) Garantir que decisões do three.js estejam refletidas nos docs
